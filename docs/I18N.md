# i18n - Internationalization System

Disapp framework includes a powerful, lightweight i18n system for multi-language support. Easily manage translations in JSON files and switch languages on the fly.

## Quick Start

### 1. Create Locale Files

Create a `locales` folder in your bot project:

```
locales/
├── en.json
├── tr.json
├── de.json
└── fr.json
```

### 2. Define Translations

**locales/en.json**

```json
{
  "commands": {
    "ping": {
      "description": "Bot latency",
      "response": "🏓 Pong! {latency}ms"
    },
    "ban": {
      "success": "✅ **{user}** has been banned.",
      "reason": "📝 Reason: {reason}"
    }
  },
  "buttons": {
    "confirm": "Confirm",
    "cancel": "Cancel"
  },
  "errors": {
    "noPermission": "❌ You don't have permission!"
  }
}
```

**locales/tr.json**

```json
{
  "commands": {
    "ping": {
      "description": "Bot gecikmesi",
      "response": "🏓 Pong! {latency}ms"
    },
    "ban": {
      "success": "✅ **{user}** yasaklandı.",
      "reason": "📝 Sebep: {reason}"
    }
  },
  "buttons": {
    "confirm": "Onayla",
    "cancel": "İptal"
  },
  "errors": {
    "noPermission": "❌ Yetkiniz yok!"
  }
}
```

### 3. Initialize i18n in Your Bot

```typescript
import { i18n } from "@disapp/core";
import path from "path";

const i18nInstance = i18n({
  defaultLanguage: "en",
  fallbackLanguage: "en",
  localesPath: path.join(__dirname, "../locales"),
});

i18nInstance.loadLocales(path.join(__dirname, "../locales"));
```

### 4. Use in Commands

```typescript
import { Command, I18nHelper } from "@disapp/core";
import { SlashCommandBuilder } from "discord.js";

export default class PingCommand extends Command {
  async execute(interaction: any) {
    const lang = interaction.locale?.split("-")[0] || "en";
    const latency = Math.round(interaction.client.ws.ping);

    const message = I18nHelper.command("ping.response", lang, { latency });
    await interaction.reply(message);
  }
}
```

## API Reference

### I18n Class

#### `getInstance(options?: I18nOptions): I18n`

Get or create the i18n singleton instance.

```typescript
const i18n = I18n.getInstance({
  defaultLanguage: "en",
  fallbackLanguage: "en",
  localesPath: "./locales",
});
```

#### `loadLocales(localesPath: string): void`

Load all JSON files from a directory.

```typescript
i18n.loadLocales("./locales");
```

#### `addLocale(language: string, translations: Record<string, any>): void`

Add translations programmatically.

```typescript
i18n.addLocale("es", {
  commands: {
    ping: { response: "🏓 ¡Pong! {latency}ms" },
  },
});
```

#### `t(key: string, language?: string, params?: Record<string, any>): string`

Translate a key with optional parameters.

```typescript
const text = i18n.t("commands.ping.response", "en", { latency: 50 });
```

#### `getLanguages(): string[]`

Get all available languages.

```typescript
const langs = i18n.getLanguages();
```

#### `hasLanguage(language: string): boolean`

Check if a language is available.

```typescript
if (i18n.hasLanguage("tr")) {
  // Turkish is available
}
```

#### `setDefaultLanguage(language: string): void`

Set the default language.

```typescript
i18n.setDefaultLanguage("tr");
```

### I18nHelper Class

Convenience methods for common translation categories. Use `t()` for any custom key:

#### `t(key: string, language?: string, params?: Record<string, any>): string`

Generic translation - works with ANY key structure.

```typescript
I18nHelper.t("commands.ping.response", "en", { latency: 50 });
I18nHelper.t("ticket.created", "en");
I18nHelper.t("random.joke", "en");
I18nHelper.t("custom.nested.key", "en", { param: "value" });
```

#### `tUser(userId: string, key: string, params?: Record<string, any>): string`

Translate using the user's preferred language (set via `setUserLanguage()`).

```typescript
// Set user's language preference
I18nHelper.setUserLanguage("user123", "tr");

// Later, translate using their language
const message = I18nHelper.tUser("user123", "commands.ping.response", {
  latency: 50,
});
// Uses Turkish if available, falls back to default language
```

#### `setUserLanguage(userId: string, language: string): void`

Set a user's preferred language for translations.

```typescript
I18nHelper.setUserLanguage("user123", "tr");
I18nHelper.setUserLanguage("user456", "de");
```

#### `getUserLanguage(userId: string): string`

Get a user's preferred language.

```typescript
const userLang = I18nHelper.getUserLanguage("user123");
console.log(userLang); // 'tr'
```

#### `button(labelKey: string, language?: string, params?: Record<string, any>): string`

Shortcut for button labels (prefixes with `buttons.`).

```typescript
const label = I18nHelper.button("confirm", "en");
```

#### `modal(titleKey: string, language?: string, params?: Record<string, any>): string`

Shortcut for modal titles (prefixes with `modals.`).

```typescript
const title = I18nHelper.modal("settings", "en");
```

#### `message(messageKey: string, language?: string, params?: Record<string, any>): string`

Shortcut for messages (prefixes with `messages.`).

```typescript
const msg = I18nHelper.message("welcome", "en");
```

#### `error(errorKey: string, language?: string, params?: Record<string, any>): string`

Shortcut for errors (prefixes with `errors.`).

```typescript
const err = I18nHelper.error("noPermission", "en");
```

#### `command(commandKey: string, language?: string, params?: Record<string, any>): string`

Shortcut for command text (prefixes with `commands.`).

```typescript
const text = I18nHelper.command("ping.response", "en", { latency: 50 });
```

#### `embed(embedKey: string, language?: string, params?: Record<string, any>): string`

Shortcut for embed content (prefixes with `embeds.`).

```typescript
const content = I18nHelper.embed("welcome", "en");
```

### Usage Examples

#### Basic Command with User Language

```typescript
import { Command, I18nHelper } from "@disapp/core";
import { SlashCommandBuilder } from "discord.js";

export default class PingCommand extends Command {
  async execute(interaction: any) {
    const userId = interaction.user.id;
    const latency = Math.round(interaction.client.ws.ping);

      latency: latency.toString(),
```

#### Language Selection Command

```typescript

  async execute(interaction: any) {
    const userId = interaction.user.id;

```

```typescript
import { Command, I18nHelper } from "@disapp/core";
import { SlashCommandBuilder } from "discord.js";

export default class GreetCommand extends Command {
  constructor() {
    super({
      name: "greet",
      description: "Greet a user",
      data: new SlashCommandBuilder()
        .setName("greet")
        .setDescription("Greet a user")
        .addUserOption((o) =>
          o.setName("user").setDescription("User to greet").setRequired(true),
        ),
      execute: async () => {},
    });
  }

  async execute(interaction: any) {
    const user = interaction.options.getUser("user");
    const lang = interaction.locale?.split("-")[0] || "en";

    const greeting = I18nHelper.t("greet.message", lang, {
      user: user.username,
    });
    await interaction.reply(greeting);
  }
}
```

### Components V2 with i18n

```typescript
import { Command, v2, I18nHelper } from "@disapp/core";
import { SlashCommandBuilder, ButtonStyle } from "discord.js";

export default class MenuCommand extends Command {
  async execute(interaction: any) {
    const lang = interaction.locale?.split("-")[0] || "en";
    const t = (key: string, params?: Record<string, any>) =>
      I18nHelper.t(key, lang, params);

    const message = v2()
      .text(`# ${t("menu.title")}`)
      .separator()
      .text(t("menu.description"))
      .buttons(
        {
          id: "confirm",
          label: I18nHelper.button("confirm", lang),
          style: ButtonStyle.Success,
          onClick: async (i) => {
            await i.reply(t("menu.confirmed"));
          },
        },
        {
          id: "cancel",
          label: I18nHelper.button("cancel", lang),
          style: ButtonStyle.Danger,
          onClick: async (i) => {
            await i.reply(t("menu.cancelled"));
          },
        },
      )
      .build();

    await interaction.reply(message);
  }
}
```

## Translation File Structure

Organize translations hierarchically. You can use ANY structure you want - there's no requirement for specific categories:

### Recommended Structure

```json
{
  "commands": {
    "ping": { "description": "...", "response": "..." },
    "ban": { "description": "...", "success": "...", "error": "..." }
  },
  "buttons": { "confirm": "...", "cancel": "..." },
  "modals": { "settings": "...", "config": "..." },
  "messages": { "welcome": "...", "goodbye": "..." },
  "errors": { "noPermission": "...", "userNotFound": "..." },
  "embeds": { "welcome": "...", "help": "..." }
}
```

### Custom Structure (Completely Valid)

You can organize however you want:

```json
{
  "ticket": {
    "created": "Ticket created!",
    "closed": "Ticket closed!"
  },
  "random": {
    "joke": "Why did the chicken cross the road?",
    "fact": "Did you know..."
  },
  "test": {
    "message": "This is a test"
  }
}
```

Then use them:

```typescript
I18nHelper.t("ticket.created", "en");
I18nHelper.t("random.joke", "en");
I18nHelper.t("test.message", "en");
```

### No Restrictions

- Use any key names you want
- Use any nesting depth
- Mix and match categories
- Create custom categories for your needs

## Parameter Substitution

Use `{paramName}` syntax for dynamic values:

```json
{
  "commands": {
    "ban": {
      "success": "✅ **{user}** has been banned.",
      "reason": "📝 Reason: {reason}",
      "duration": "⏱️ Duration: {duration} days"
    }
  }
}
```

```typescript
const message = I18nHelper.command("ban.success", "en", {
  user: "John",
  reason: "Spam",
  duration: 7,
});
```

## Best Practices

1. **Organize by category** - Use nested objects for better organization
2. **Use consistent naming** - Follow a naming convention (e.g., `commands.ban.success`)
3. **Provide fallback language** - Always have English as fallback
4. **Use parameters** - Avoid hardcoding values in translations
5. **Keep translations updated** - Maintain all language files together
6. **Use descriptive keys** - Make keys self-documenting
7. **Test all languages** - Verify translations work correctly

## Performance

- **Lazy loading** - Locales are loaded on demand
- **Caching** - Translations are cached in memory
- **Fast lookup** - O(1) key access using nested objects
- **No external dependencies** - Pure JavaScript implementation

## Limitations

- Maximum key depth: Unlimited (but keep reasonable)
- Parameter names: Alphanumeric and underscores only
- File format: JSON only
- Encoding: UTF-8 required

## Migration from Hardcoded Strings

```typescript
const lang = interaction.locale?.split("-")[0] || "en";
```
